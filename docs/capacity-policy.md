# 容量策略

> GitHub Pages 只做网站外壳和轻量知识索引，不能当资料仓库。

---

## GitHub Pages 硬限制

| 限制项 | 阈值 | 说明 |
|--------|------|------|
| 仓库建议大小 | ≤ 1GB | 超过后 GitHub 可能发警告 |
| 发布网站大小 | ≤ 1GB | 超过后网站可能无法正常部署 |
| 月带宽（软限制） | ~100GB | 超过后 GitHub 可能限流 |
| 单文件警告 | > 50MiB | Git 会提示但不阻止 |
| 单文件阻止 | > 100MiB | GitHub 拒绝推送 |
| .gitignore 忽略的文件 | 不占仓库空间 | 已配置，见根目录 .gitignore |

来源：[GitHub Pages limits](https://docs.github.com/en/pages/getting-started-with-github-pages/github-pages-limits)

## 三层存储架构

```
GitHub Pages     →  展示层：网页 + 轻量数据
本地 D:\Sy课程开发  →  原始资料库：所有源文件
网盘 / 对象存储     →  大文件和会员资源
```

### 第一层：GitHub Pages（放什么）

**适合上传的内容：**

| 类型 | 示例 | 单文件典型大小 |
|------|------|--------------|
| HTML | 页面文件 | 5-50K |
| CSS | 样式表 | 10-20K |
| JS | 脚本 | 5-50K |
| Markdown | 周报源文件、文档 | 5-20K |
| YAML / JSON | 工具数据、结构化索引 | 5-50K |
| SVG | 图标、favicon | < 5K |
| 小图片 | 页面配图、logo | < 200K/张 |

**规则：** .gitignore 已配置为自动忽略以下类型，即使 `git add .` 也不会上传：

```
*.pptx *.ppt *.docx *.doc *.xlsx *.xls  # Office 文档
*.pdf                                   # PDF
*.mp4 *.mov *.avi *.mkv *.wav *.mp3     # 视频/音频
*.zip *.rar *.7z *.tar *.gz             # 压缩包
*.psd *.ai *.sketch                     # 设计文件
*.iso *.dmg *.exe                       # 其他
```

### 第二层：本地资料库（放什么）

`D:\Sy课程开发\` 是所有原始资料的主仓库，包括：

- 课程PPT、课件包
- 项目管理模板（Excel/Word）
| 培训视频录像
| 课程照片、高清图片
| 客户案例资料
| 内容资产登记表（Excel/CSV）

**规则：** 这些文件不上传 GitHub，只在本地维护。通过内容资产登记表建立索引，网站只展示目录和摘要。

### 第三层：网盘 / 对象存储（放什么，未来接入）

需要在线访问但体积过大的文件：

| 文件类型 | 建议存储 | 说明 |
|---------|---------|------|
| 会员版周报 PDF | 对象存储 | 需要权限控制 |
| 视频教程 MP4 | 对象存储 / 视频平台 | 体积大，适合CDN分发 |
| 课件下载包 ZIP | 对象存储 / 网盘 | 需要权限控制 |
| 课程完整资料 | 网盘 | 分享链接即可 |

**可选方案（按成本排序）：**

| 方案 | 免费额度 | 适合 |
|------|---------|------|
| 百度网盘 / 腾讯微云 | 免费但限速 | 分享不频繁的文件 |
| Cloudflare R2 | 10GB免费 + 零出口流量费 | 国际访问，推荐 |
| 腾讯云 COS | 50GB免费6个月 | 国内访问快 |
| Supabase Storage | 1GB免费 | 需要登录权限的场景 |

## 内容资产登记表中的"存放位置"字段

内容资产登记表（content-inventory）中每个文件必须标注实际存放位置：

| 字段 | 示例 |
|------|------|
| 文件名 | AI赋能项目管理课件.pptx |
| 类型 | PPT |
| 主题 | AI / 项目管理 / 企业培训 |
| 公开级别 | 会员 |
| **存放位置** | D:\Sy课程开发\2026年XX月XX日\（本地） |
| **网盘链接** | https://pan.baidu.com/... （如有） |
| **网站展示** | 只显示名称+简介，不嵌入原文件 |

## 容量检查方法

在 VS Code 终端中执行（PowerShell 用 `;` 分隔）：

```bash
# 查看网站文件总大小（排除 .git 和 .workbuddy）
du -sh . --exclude=.git --exclude=.workbuddy

# 查看各文件大小排序
find . -not -path "./.git/*" -not -path "./.workbuddy/*" -type f -exec ls -lh {} \; | awk '{print $5, $NF}' | sort -rh | head -20
```

**安全线：** 网站文件总量保持在 **500MB 以下**（1GB 限制的一半），留足余量。

**警告线：** 超过 **800MB** 时需要清理，将大文件迁移到对象存储。
