# 网站建设与运营原则

这份文档用于指导个人知识产品网站的建设、内容整理、免费/会员设计、上线节奏与长期运营。

## 1. 网站定位原则

这个项目不只是一个个人主页，而是一个长期运营的个人知识产品平台。

核心定位：

```text
个人品牌官网
+ 项目管理课程平台
+ AI工具资源库
+ 课程资料下载中心
+ 会员知识服务系统
```

网站的真正价值不在页面本身，而在于把多年积累的课程、PPT、文章、模板、案例、AI工具周报和教学经验整理成可展示、可搜索、可复用、可转化的知识资产。

## 2. 内容资产优先原则

先整理内容，再设计页面。

建站前要先回答：

```text
我有哪些资源？
这些资源属于哪些主题？
适合哪些人？
哪些内容可以公开？
哪些内容适合会员？
哪些内容适合变成课程、文章、模板或工具卡片？
```

不要只按栏目整理内容，而要把每个资源当成一个可以多处复用的知识资产。

例如：

```text
AI赋能项目管理PPT
```

可以同时属于：

```text
PPT
课程资料
项目管理
AI
企业培训
会员资源
```

这样未来可以自动出现在课程页、下载中心、AI专题页和会员资源页中。

## 3. 免费与会员设计原则

不要把免费和会员简单设计成“免费只能看一点，会员才能看全部”。

更好的设计是：

```text
免费：告诉用户是什么
会员：告诉用户怎么做
```

例如 AI 工具：

免费用户可以看到：

```text
工具名称
官网链接
一句话介绍
适用场景
推荐指数
```

会员用户获得：

```text
最佳入口
注册方法
国内可用情况
使用技巧
Prompt
教学案例
企业应用建议
替代方案
注意事项
视频教程
```

会员价值不应是“藏链接”，而应是提供经验、效率、方法、案例和可直接使用的资料。

## 4. 官网链接公开原则

AI工具、课程资源或公开资料的基础链接，原则上不建议只给会员。

免费用户应该能看到官网、基础介绍和使用方向。

会员内容应集中在：

```text
节省时间
降低试错成本
提供可复制的方法
提供案例和模板
提供持续更新
提供专属整理和解释
```

这样用户更容易接受，也更容易形成信任。

## 5. 第一版网站从小做起原则

第一版网站不要一上来就做完整会员系统、支付系统和复杂后台。

第一版目标是：

```text
能展示个人品牌
能展示核心课程
能展示免费资源
能让用户联系你
能承载后续扩展
```

当前最适合先上线的内容：

```text
个人介绍
情绪陪伴 · 禅修分享（佛学14年 / 九型人格 / 中文服务）
AI赋能项目管理课程介绍
教师免费AI工具周报
联系合作方式
```

第一版先做清楚，比一开始做大更重要。

## 6. 网站栏目克制原则

第一版导航不要太复杂。

建议先用：

```text
首页
关于我
AI赋能项目管理
教师AI工具周报
联系合作
```

等内容增加后，再扩展为：

```text
课程中心
AI资源中心
知识库
下载中心
企业解决方案
会员中心
```

先让用户看懂，再逐步丰富。

## 7. 内容格式标准化原则

不要长期维护大量零散 HTML 文件。

已有 HTML 可以先作为素材保留，但后续建议逐步转成：

```text
Markdown
JSON
YAML元数据 + Markdown正文
```

每篇内容都尽量包含：

```text
标题
摘要
分类
标签
适合对象
更新时间
公开级别
正文
相关资源
```

这样后续可以自动生成文章页、专题页、课程页和资源卡片。

## 8. 搜索优先原则

对于资料量很大的知识平台，搜索比栏目更重要。

理想体验：

```text
搜索：风险登记册

返回：
相关文章
PPT课件
Excel模板
课程章节
企业案例
AI工具
会员下载
```

第一阶段可以先不做复杂搜索，但整理资料时要提前保留分类、标签和关键词。

## 9. AI工具导航重点打造原则

AI工具导航是未来最有流量潜力和更新价值的栏目。

每个工具建议有统一卡片：

```text
工具名称
一句话介绍
官网链接
适合对象
适用场景
推荐指数
是否免费
是否国内可用
同类替代工具
更新时间
```

长期可扩展为：

```text
教师AI
项目管理AI
国产AI
办公AI
PPT工具
视频工具
Agent工具
Prompt工具
```

## 10. 课程转化原则

课程页面不是简单介绍课程，而是要帮助用户判断“这门课是否适合我”。

每门课程建议包含：

```text
课程名称
一句话定位
适合对象
能解决的问题
课程收益
课程大纲
授课形式
典型案例
可交付资料
合作咨询方式
```

对于企业客户，最好增加“企业解决方案”栏目，用行业痛点来组织内容。

## 11. 企业客户导向原则

企业客户通常不是按课程名找服务，而是按问题找解决方案。

建议逐步建设：

```text
制造业项目管理数字化能力提升方案
高校教师AI教学能力提升方案
企业PMO建设与项目治理方案
AI赋能办公效率提升方案
```

每个方案页面可以包含：

```text
行业痛点
培训需求
课程组合
交付成果
适合岗位
相关案例
合作方式
```

## 12. 安全与会员资源原则

公开内容可以放在 GitHub 或静态网站中。

会员内容、PPT、PDF、视频、模板等，不建议直接放在公开 GitHub 仓库里。

后续可考虑：

```text
Cloudflare R2
Supabase Storage
其他对象存储
```

通过登录权限和签名下载来保护会员资源。

## 13. 技术选型原则

第一阶段优先简单、稳定、低成本。

推荐：

```text
Astro
Markdown
GitHub仓库
Cloudflare Pages 或 GitHub Pages
```

如果只是公开静态网站，GitHub Pages 足够。

如果未来要登录、会员、权限、表单、下载保护，Cloudflare Pages 更适合长期扩展。

## 14. 三阶段推进原则

### 第一阶段：公开静态网站

目标：

```text
首页
个人介绍
AI赋能项目管理
教师AI工具周报
联系合作
```

重点是先上线、先能看、先能传播。

### 第二阶段：资源库和搜索

目标：

```text
内容资产登记表
AI工具导航
下载中心
标签系统
站内搜索
资源卡片
```

重点是把资料变成可查找、可复用的知识资产。

### 第三阶段：会员与AI知识库

目标：

```text
登录注册
会员内容
付费下载
课程资料库
AI问答
AI助教
个性化推荐
```

重点是把网站升级为长期运营的个人知识操作系统。

## 15. 当前执行顺序

当前建议先做：

```text
1. 建立首页
2. 建立个人介绍页面
3. 建立 AI赋能项目管理 页面
4. 建立 教师AI工具周报 活动页
5. 把原来的 teacher-ai-weekly-report.html 挂接进去
6. 加上联系合作入口
7. 后续再整理资源库和会员系统
```

第一版不要追求“大而全”，而要追求“清楚、有用、能上线”。

## 16. 一句话总原则

先把你的知识资产整理成清楚的公开入口，再逐步增加搜索、资源库、会员和AI助教。网站不是终点，而是你个人知识产品长期运营的起点。
